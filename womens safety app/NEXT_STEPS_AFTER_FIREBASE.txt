================================================================================
TROUBLESHOOTING
================================================================================

Problem: "google-services.json not found"
Solution: Download from Firebase Console (see STEP 1)

Problem: "Flutter command not found"  
Solution: Install Flutter from https://flutter.dev/docs/get-started/install/windows
          Add to PATH: C:\src\flutter\bin (or your Flutter location)

Problem: "No devices found"
Solution: - Enable USB debugging on phone
          - Try different USB cable/port
          - Install phone drivers from manufacturer website
          - Use "flutter doctor" to check setup

Problem: Build errors with Firebase
Solution: Run these commands:
          flutter clean
          cd android
          .\gradlew clean
          cd ..
          flutter pub get
          flutter run

Problem: App crashes on launch
Solution: Check that google-services.json is in correct location:
          android\app\google-services.json (NOT android\ folder!)

Problem: "Gradle build failed"
Solution: Check internet connection (Gradle downloads dependencies)
          Wait and retry - first build needs to download a lot

================================================================================
WHAT TO DO IF FIREBASE SERVICES NOT WORKING
================================================================================

If you get Firebase errors after setup, verify on Firebase Console:

1. Authentication Enabled:
   - Firebase Console ‚Üí Authentication ‚Üí Sign-in method
   - Enable "Email/Password"

2. Firestore Database Created:
   - Firebase Console ‚Üí Firestore Database
   - Should show "Create database" or existing database
   - If not created, click "Create database" ‚Üí Start in test mode

3. Cloud Messaging Enabled:
   - Firebase Console ‚Üí Cloud Messaging
   - Should show "Cloud Messaging API (Legacy)" enabled

4. Security Rules Set:
   - Firebase Console ‚Üí Firestore Database ‚Üí Rules
   - Should allow read/write for authenticated users
   - If not, set rules to:

     rules_version = '2';
     service cloud.firestore {
       match /databases/{database}/documents {
         match /{document=**} {
           allow read, write: if request.auth != null;
         }
       }
     }
